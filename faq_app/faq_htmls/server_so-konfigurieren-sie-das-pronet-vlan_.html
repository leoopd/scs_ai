      So konfigurieren Sie das ProNet VLAN


      In diesem Artikel beschreiben wir ein, an erfahrene Nutzer gerichtetes Feature für dedizierte Root-Server Linux und Windows (mit dem Paketnamen Dxxx). Das ProNet VLAN steht bei Root-Servern mit dem Paketnamen CPxxx nicht zur Verfügung.

      Was ist STRATO ProNet VLAN?

      Das ProNet VLAN dient der Kommunikation zwischen zwei oder mehr Servern über eine zweite Netzwerkinfrastruktur. Diese ist unabhängig von der primären Netzwerkinfrastruktur, über die Ihre Server mit dem Internet verbunden sind. Einer Ihrer Server kann de facto komplett auf die externe Anbindung verzichten und nur über den anderen Server erreichbar sein. ProNet VLAN ist aktuell nur für dedizierte Root-Server (Linux/Windows) einsetzbar.

      Jeder Server, für den Sie das Produkt ProNet VLAN gebucht haben, wird über eine zweite Ethernetschnittstelle an unsere zweite Netzwerkinfrastruktur angebunden.


      So bestellen Sie ProNet VLAN

      Loggen Sie sich zunächst in Ihren passwortschützten STRATO Kunden-Login ein. Dort klicken Sie in der Übersicht Ihrer Pakete auf Einstellungen bei Ihrem Serverpaket, um in Ihr Server-Login zu gelangen.In dieser wählen Sie unter dem Menüpunkt Netzwerk & Last die Option ProNet VLAN.


      Die Bereitstellung erfordert die Installation durch einen Techniker in unserem Rechenzentrum und kann wenige Stunden in Anspruch nehmen.

      Es stehen verschiedene Bandbreiten für Ihre Server von 10 Mbit/s bis 1000 Mbit/s zur Auswahl. Die Bandbreite kann bei den jeweiligen Servern unterschiedlich sein. Über den Link Upgrade können Sie höhere Bandbreiten in den Schritten 10, 100, 200, 500 und 1000 Mbit/s buchen.

      Zusätzlich kann ein virtuelles LAN (VLAN) eingerichtet werden. Um ein VLAN aufzubauen, benötigt jeder Ihrer Server einen Connect. Ist dieser aktiviert, kann der Server als VLAN User in ein bestehendes VLAN eingebunden werden.

      Über VLAN: Mit der Option VLAN erhalten Sie eine eindeutige VLAN ID. Zusätzlich erhalten Sie ein privates IP-Netz aus dem 10/8-Präfix nach RFC 1918, beispielsweise 10.31.1.0/24. Die ersten Adressen Ihres IP-Netzes 10.31.1.1 - 10.31.1.7 sind für Infrastrukturkomponenten reserviert.


      Wie richte ich ProNet VLAN ein?

      Die Konfiguration Ihres VLAN führen Sie über den STRATO Server-Login durch.

      Ist das VLAN aktiv, kann begonnen werden weitere VLAN-User einzubinden.

      Entweder lädt der VLAN-Owner den User über den STRATO Server-Login ein, oder der User fragt beim Owner eine Mitgliedschaft an. In beiden Fällen muss die Gegenseite die Mitgliedschaft bestätigen.



      Über den Button Mitgliedschaft anbieten können Sie Server zu Ihrem VLAN hinzufügen.

      • Wenn Sie für den hinzuzufügenden Server über das Passwort zum STRATO Server-Login verfügen, so können Sie dieses angeben. Der Server wird dann ohne Umweg Mitglied in Ihrem VLAN.
      • Sind Sie nicht im Besitz des Passworts, weil Sie beispielsweise den Server eines anderen Kunden in Ihr VLAN aufnehmen möchten, so wird im STRATO Server-Login dieses Servers Ihre Anfrage zur Annahme oder Ablehnung vorgelegt.


      VLAN-Typen

      Es existieren zwei VLAN-Typen, Untagged VLANs und Tagged VLANs.

      Untagged VLANDas erste auf einem Server eingerichtete VLAN wird immer in dieser Konfiguration betrieben. Die VLAN-Funktionalität wird dabei von der ProNet-Infrastruktur verwaltet.
      Tagged VLANWerden zusätzliche VLANs auf einem Server eingerichtet, so ist die Konfiguration als Tagged VLAN erfordrlich.


      Wie konfiguriere ich meine Server für die Verwendung des ProNet VLAN?

      Um Ihr ProNet VLAN nutzen zu können, müssen Sie auf Ihren Servern das zweite Netzwerkinterface statisch konfigurieren.


      Temporäre Konfiguration unter Linux (distributionsabhängig)

      Diese Art der Konfiguration ist nicht bootfest. Um sicherzustellen, dass das ProNet VLAN nach einem Neustart Ihres Servers weiterhin funktioniert, folgen Sie bitte der weiter unten für das von Ihnen genutzte Betriebssystem angegebenen Konfigurations-Anleitung.

      Prüfen Sie, ob Ihr zweites Netzwerkinterface mit dem ProNet VLAN-Switch verbunden ist.

      h123456:~ # ethtool eth1
      Settings for eth1:
      Supported ports: [ TP ]
      Supported link modes: 10baseT/Half 10baseT/Full
      100baseT/Half 100baseT/Full
      1000baseT/Full
      Supports auto-negotiation: Yes
      Advertised link modes: 10baseT/Half 10baseT/Full
      100baseT/Half 100baseT/Full
      1000baseT/Full
      Advertised auto-negotiation: Yes
      Speed: 10Mb/s
      Duplex: Full
      Port: Twisted Pair
      PHYAD: 1
      Transceiver: internal
      Auto-negotiation: on
      Supports Wake-on: umbg
      Wake-on: g
      Current message level: 0x00000007 (7)
      Link detected: yes
      Speedklassen: 200 Mbps und 500 Mbps werden als 1 Gbps angezeigt, da Limitierung auf Switchseite stattfindet.

      Entscheidend ist in dieser Ausgabe die Zeile: Link detected: yes

      Diese Zeile signalisiert eine bestehende Verbindung.

      Wird hier keine Verbindung angezeigt, kann dies daran liegen, dass sich das Netzwerkinterface eth1 im Power-Down-Zustand befindet. Aktivieren Sie eth1 gegebenenfalls mit dem Befehl:

      h123456:~ # ip addr add 10.x.y.z/24 dev eth1

      und prüfen Sie die Verbindung anschließend erneut.

       
      Untagged VLAN

      h123456:~ # ifconfig eth1 10.x.y.z netmask 255.255.255.0 up

      Verwenden Sie bitte anstatt der 10.x.y.z bitte die IP-Adresse Ihres VLANs. Diese finden Sie im STRATO Server-Login Ihres Servers unter dem Menüpunkt Netzwerk & Last / ProNet VLAN / interne IP-Adresse.

      Tagged VLAN

      h123456:~ #ip link add link eth1 name eth1.#### type vlan id ####
      h123456:~ # ip addr add 10.x.y.z/24 brd 10.x.y.255 dev eth1.####
      h123456:~ # ip link set eth1.#### up

      Ersetzen Sie #### durch die entsprechende ID Ihres tagged VLANs. Anstatt der 10.x.y.z verwenden Sie bitte die IP-Adresse Ihres VLANs.
      Die Angaben finden Sie in Ihrem STRATO Server-Login unter dem Menüpunkt Security /ProNet VLAN / VLAN und unter dem Menüpunkt Netzwerk & Last / ProNet VLAN / interne IP-Adresse.


      Permanente Konfiguration unter Linux (distributionsabhängig)


      Debian und Ubuntu

      Untagged VLAN


      Editieren Sie die Datei /etc/network/interfaces
      Passen Sie den Abschnitt für das Netzwerkinterface eth1 an, um eine IP-Adresse dem zweiten Netzwerkinterface zuzuordnen:

      auto eth1
      iface eth1 inet static
      address 10.x.y.z
      netmask 255.255.255.0
      network 10.x.y.0
      broadcast 10.x.y.255
      mtu 1500
      Starten Sie anschließend die Netzwerkkonfiguration neu:
      h123456:~ # ifdown eth1; ifup eth1


      Tagged VLAN

      h123456:~ # apt-get install vlan

      Editieren Sie die Datei /etc/network/interfaces
      Fügen Sie einen Abschnitt für Ihr tagged VLAN hinzu:
      Ersetzen Sie #### durch die entsprechende ID Ihres tagged VLANs.
       

      # VLAN ####
      auto vlan####
      iface vlan#### inet static
      address 10.x.y.z
      netmask 255.255.255.0
      network 10.x.y.0
      broadcast 10.x.y.255
      mtu 1500
      vlan_raw_device eth1
      Starten Sie anschließend die Netzwerkkonfiguration neu:
      h123456:~ # ifdown vlan####; ifup vlan####


      CentOS 7

      Untagged VLAN

      Editieren Sie die Datei /etc/sysconfig/network-scripts/ifcfg-eth1 und passen Sie den Inhalt entsprechend der folgenden Angaben an:

      TYPE=Ethernet
      DEVICE=eth1
      BOOTPROTO=static
      IPADDR=10.x.y.z
      NETMASK=255.255.255.0
      USERCTL=yes
      ONBOOT=yes

      Starten Sie anschließend die Netzwerkkonfiguration neu:

      h123456:~ # service network restart

       
      Tagged VLAN

      Legen Sie die Datei /etc/sysconfig/network-scripts/ifcfg-vlan#### mit folgendem Inhalt an:

      Ersetzen Sie #### durch die entsprechende ID Ihres tagged VLANs.

      VLAN=yes
      VLAN_NAME_TYPE=VLAN_PLUS_VID_NO_PAD
      DEVICE=vlan####
      PHYSDEV=eth1
      BOOTPROTO=static
      ONBOOT=yes
      TYPE=Ethernet
      IPADDR=10.x.y.z
      NETMASK=255.255.255.0

      Starten Sie anschließend die Netzwerkkonfiguration neu:

      h123456:~ # ifdown vlan####; ifup vlan####


      Windows 2012

      Bitte aktivieren Sie zunächst das zweite Netzwerkinterface.

      Öffnen Sie hierzu über das Startmenü die Systemsteuerung. Klicken sie anschließend auf Hardware und öffnen Sie unter Geräte und Drucker den Geräte-Manager.

      Lassen Sie sich die installierten Netzwerkinterfaces durch einen Klick auf das Symbol vor Netzwerkadapter anzeigen.

      Ein deaktiviertes Netzwerkinterface erkennen Sie an einem Symbol mit einem Pfeil, der nach unten zeigt. Aktivieren Sie dieses Interface durch Rechtsklick Aktivieren oder über Aufruf der Eigenschaften dieses Interfaces.

      Untagged VLAN


      Taskbar Systray, Rechtsklick auf Symbol für Netzwerkverbindungen / Netwerk- und Freigabecenter öffnen


      Klick auf Ethernet 2 unter aktive Netzwerke anzeigen


      Klicken Sie anschließend auf Eigenschaften


      Wählen anschließend im Fenster Eigenschaften von Ethernet 2 den Punkt Internetprotokoll Version 4 und klicken Sie erneut auf Eigenschaften.

      Im Fenster Eigenschaften geben Sie die folgenden Punkte an:

      Folgende Einstellungen verwenden:
      IP-Adresse: 10.x.y.z
      Subnetzmaske: 255.255.255.0
      Standardgateway: 10.x.y.1

      Ersetzen Sie den Platzhalter 10.x.y.z bitte durch die interne IP-Adresse und den Platzhalter 10.x.y.1 durch die IP-Adresse des Gateways ihres VLANs. Diese finden Sie in Ihrem STRATO Server-Login unter: Security / ProNet VLAN.

      Tagged VLAN


      Um ein tagged VLAN auf Ihrem Windows 2012 Server zu installieren, öffnen Sie bitte zunächst den Server-Manager.


      Rechtsklicken Sie bitte unter Alle Server/Server auf den Eintrag mit dem Hostnamen Ihres Servers und anschließend auf NIC-Teamvorgang konfigurieren.


      Klicken Sie im Fenster NIC-Teamvorgang unter Adapter und Schnittstellen einen Rechtsklick auf den Eintrag Ethernet 2 und klicken Sie anschließend auf Zum neuen Team hinzufügen.

      Erstellen Sie anschließend ein neues Team für den Adapter Ethernet 2 indem Sie einen Namen für das Team eingeben und den Vorgang mit OK abschließen. Wählen Sie hierfür jedoch im Netzwerk- und Freigabecenter die eben erstellte VLAN-Verbindung aus.

      Klicken Sie anschließend bitte im Feld Adapter und Schnittstellen auf Teamschnittstellen.


      Mit einem Rechtsklick auf das neuerstellte Team und anschließendem Klick auf Eigenschaften gelangen Sie zu den Netzwerkadaptereigenschaften.

      Bitte klicken Sie unter VLAN-Mitgliedschaft auf Spezifisches VLAN und geben Sie die VLAN-ID für Ihr tagged VLAN ein.


      Die ID Ihres VLANS finden Sie im STRATO Server-Login unter dem Menüpunkt Security / ProNet VLAN / VLAN.

      Bitte weisen Sie dem soeben erstellten Interface anschließend eine IP-Adresse zu. Folgen Sie hierzu der oben unter untagged VLAN beschriebenen Vorgehensweise. Wählen Sie hierfür jedoch das neuerstellte VLAN-Interface im Netzwerk- und Freigabecenter aus.


      Die Informationen zur IP-Adresse Ihres tagged VLANs finden Sie im STRATO Server-Login unter dem Menüpunkt Security / ProNet VLAN / interne IP-Adresse


      Hinweis: Bitte achten Sie auf die Konfiguration der Windows-Firewall Ihres Servers, da diese den Zugriff auf das ProNet VLAN blockieren kann.


      Einsatzszenarien

      High-Availability-Lösung
      Zwei Server mit hauptsächlich statischen Webseiten werden zur Erhöhung der Verfügbarkeit gegenseitig gespiegelt. Für ein Failover-Szenario wird die STRATO ClusterIP eingesetzt. Zwischen den Servern dient eine 10 MBit Verbindung für den Heartbeat und den regelmäßigen Datenabgleich.

      Caching-Webproxy-Lösung
      Ein Webserver wird über das ProNet VLAN mit einem zweitem System verbunden, welches als Caching Webproxy arbeitet. Vorteile sind deutlich bessere Performance und höhere Sicherheit. Je nach Bedarf kommt eine Anbindung mit 100 MBit oder mehr zum Einsatz.

      Front-End/Back-End-Lösung
      Ein Webserver wird über das ProNet VLAN mit einem zweiten System verbunden, welches als Datenbankserver arbeitet und ggf. andere Dienste anbietet, die nicht extern verfügbar sein müssen (z. B. User-Authentifizierung, Fileservice, Syslog). Vorteile sind auch hier einerseits deutlich bessere Performance, aber auch höhere Sicherheit, da Dienste auf getrennten Systemen laufen.

      Das Sicherheitskonzept für jedes System kann individuell an die gegebenen Anforderungen angepasst werden, ohne Kompromisse einzugehen.
      Der Datenbank Server kann ggf. vollständig vor dem Zugriff von extern abgeschirmt werden.

      Je nach Bedarf kommt eine Anbindung mit 100 MBit oder mehr zum Einsatz.

      Firewall-Gateway-Lösung
      Ein Server fungiert als dedizierter Firewall-Server. Alle anderen Server kommunizieren ausschließlich über das ProNet VLAN mit dem Firewall-Server oder untereinander.

      Front-End/Back-End-Lösung mit mehreren Servern
      Kunden mit mehreren Servern bei STRATO können auch mehr als zwei Server innerhalb eines VLANs miteinander verbinden. Ein mögliches Szenario könnte sein:
      Zwei Datenbank-Server mit MySQL Replikation werden mit zwei redundant ausgelegten Web-Servern miteinander verbunden. Die Datenbank-Server werden mit 200 MBit angeschlossen, die Web-Server jeweils mit 100 MBit. Die Datenbank-Server werden ausschließlich über Remote Console administriert und sind über das ProNet VLAN zu erreichen. Das externe Interface kann komplett deaktiviert werden, was maximale Sicherheit vor Angriffen von außen bietet.
